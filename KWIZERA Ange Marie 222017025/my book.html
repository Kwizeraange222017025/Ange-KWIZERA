        7. Attributes of employee Table
        Here are the descriptions or purposes of the attributes/columns used in employee table:

        Employee_id (primary key): id for the employee.

        First_name(varchar): first name of employee.

        Last_name(varchar): last name of employee.

        Date_of_birth(varchar): date of birth of employee.

        Attributes of payroll Table
        Here are the descriptions or purposes of the attributes/columns used in payroll table:

        Payroll_Id: The column which are represents id of payroll .
        AUTO_INCREMENT means that the database will automatically generate a unique ID for each new record, typically incrementing the previous value of payroll table.

        Employee _id(primary key): the column which are represents Employee _id.(primary key)


        Pay_period_start_date(varchar): The column which are represents the date of starting period.

        Pay_period_end_date(varchar): The columns which are represents the date of ending period.

        Attributes of tax Table
        Here are the descriptions or purposes of the attributes/columns used in tax table:

        tax_Id(primary key): The column which are represents id of tax payer which is unique value called PRIMARY KEY. It is set to auto-increment (AUTO_INCREMENT).
        AUTO_INCREMENT means that the database will automatically generate a unique ID for each new record, typically incrementing the previous value of tax table.


        Tax_name(varchar): The column which are represents name of tax to be paid.


        Tax_rate(varchar): The column which are represents the rate of tax to be paid.


        Tax_ category(varchar): The column which are represents category of tax to be paid


        Attributes of Time tracking Table
        Here are the descriptions or purposes of the attributes/columns used in Time tracking table:

        Time_id(primary key): The column which are represents time id which is unique value called PRIMARY KEY. It is set to auto-increment (AUTO_INCREMENT).
        AUTO_INCREMENT means that the database will automatically generate a unique ID for each new record, typically incrementing the previous value of time tracking table.


        Employee_id(primary key): The column which are represents id of employee which is references to the product table value called FOREIGN KEY. It is a column of time tracking table which has relationship between time tracking table.


        Clock_in_time(varchar): The column which are represents the time employee entered.


        Clock_out_time(varchar): The column which are represents the time employee loged out


        Attributes of users Table
        Here are the descriptions or purposes of the attributes/columns used in users table:

        User_ id (primary key): The column which are represents id of users which is unique value called PRIMARY KEY. It is set to auto-increment (AUTO_INCREMENT).
        AUTO_INCREMENT means that the database will automatically generate a unique ID for each new record, typically incrementing the previous value of users table.


        Username(varchar): The column which are represents name of the users.


        Password(varchar): The column which are represents the password of users.


        User_type: The column which are represents types of users.
